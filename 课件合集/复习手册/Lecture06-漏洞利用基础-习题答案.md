6. A。API 自搜索用于动态定位当前进程中的 API 入口地址。

7. 对。有漏洞不代表已有成熟利用，但 exploit 通常以某种漏洞为基础。

8. 对。Shellcode 不一定真的打开 shell。

9. 对。传统代码植入需要写入、劫持和执行权限三个条件配合。

## B. 理解应用题答案

15. A。Exploit、payload、shellcode 分别关注触发控制、攻击效果和机器码实现。

16. A。`verify` 返回 0 表示验证通过，覆盖 `flag` 为 0 可影响认证结果。

17. A。函数参数需要的是字符串地址，不是字符串内容本身。

18. A。ASLR 和系统差异会导致 API 运行时地址变化。

19. A。编码后的主体需要解码器在运行时还原。

20. 因为崩溃只是异常现象，还要看输入是否可控、覆盖目标是否有价值、控制流是否可劫持、地址和权限是否能满足利用条件。

21. 覆盖相邻变量主要改变程序数据或逻辑状态，例如认证标志；覆盖返回地址则改变函数返回后的执行位置，属于控制流劫持。

22. 因为 shellcode 通常被塞进有限输入空间，不能依赖固定加载地址，还会经过字符串函数、协议字段或过滤器处理，坏字符可能截断或破坏机器码。

23. `GetProcAddress` 能在当前运行环境中解析真实函数地址，适应 ASLR、系统版本和补丁差异；写死地址只适合很特定的环境。

## C. 综合融会题答案

24. `buffer` 只有 44 字节，`strcpy` 不检查 `code` 长度，输入过长会写出 `buffer`。

25. C 字符串拷贝会写入结尾 `\0`，如果 44 字节刚好填满 `buffer`，这个结尾字节可能写到相邻的 `flag`。

26. 字符 `'0'` 的字节值是 `0x30`，四个 `'0'` 是 `0x30303030`，不是整数 0。

27. 如果 `flag` 原值不是 1，只把最低字节写成 0 未必能让整个 4 字节整数变成 0。

28. 需要先写过 `buffer[44]`、`flag[4]` 和 `old EBP[4]`。

29. 大致是第 53 到第 56 字节，按 1 起始计数。

30. 函数返回时，`ret` 可能跳到 buffer 中，把其中字节当作机器指令执行。

31. 清零 `EBX`，后续用作 NULL 或 0 参数。

32. 保存 `"westwest"` 字符串在栈上的首地址。

33. 依次是 `uType=0`、`lpCaption="westwest"`、`lpText="westwest"`、`hWnd=NULL`。

34. 因为 DLL 加载基址会受 ASLR、系统版本和补丁差异影响，函数偏移或加载地址也可能变化。
