## 一、选择题答案

### A. 概念辨析

1. A。CVE 给具体漏洞实例编号（像身份证号），CWE 给漏洞类型分类（像病种分类）。B 顺序颠倒；C 二者都不是评分系统（CVSS 才是）；D CWE 不是数据库。

2. B。补丁已公开但大量系统未修补，属于 1day 阶段。A 0day 是漏洞尚未公开；C 已普遍修复则风险降低；D 评分结果是 CVSS 的职责。

3. A。肉鸡通常依赖"软件漏洞 + 木马"形成：漏洞提供入侵方式，木马提供长期控制。B 肉鸡可以是任何联网设备；C 肉鸡不一定要依赖蠕虫；D 肉鸡可以通过清除木马和修复漏洞来处置。

4. A。渗透测试是授权范围内的风险验证，漏洞扫描是自动化检测。B 渗透测试不只是扫描；C 扫描结果不能替代渗透测试报告；D 渗透测试必须有授权。

5. A。黑箱测试从外部视角开始，隐秘测试强调测试存在不被发现。B 黑箱测试不需要源码；C 隐秘测试也关注检测响应能力；D 二者是不同的测试方式。

### B. 堆栈与溢出

6. A。buf 占 12 字节，saved EBP 占 4 字节，返回地址占 4 字节。20 字节输入 = 12(buf) + 4(saved EBP) + 4(返回地址)，刚好覆盖全部。B 错误：返回地址被完整覆盖，不是"低 4 字节"；C 错误：saved EBP 会被覆盖；D 错误：20 > 12，会越界。

7. B。`malloc` 返回的指针指向块身，供程序直接使用。A 块首是堆管理器使用的管理信息；C 堆表入口是索引结构；D 页起始地址是操作系统管理粒度。

8. A。栈溢出常威胁返回地址（控制流），堆溢出常威胁相邻对象或堆管理元数据。B 栈溢出也能覆盖返回地址；C 栈和堆的利用方式不同；D 堆溢出通常比栈溢出更复杂。

9. A。`call` 压入返回地址并跳转，被调用函数建立栈帧，`ret` 从栈中取返回地址并跳回。B `call` 不建立栈帧；C `call` 和 `ret` 都涉及栈操作；D 返回地址由 `call` 自动压入。

10. C。`xor eax, eax` 将 EAX 与自身异或，结果为 0，不依赖其他寄存器。A 依赖 EBX 的值；B 依赖 EBX 的值；D `cmp eax, eax` 只影响标志位，不修改 EAX。

11. D。`[EBX+ESI+1000h]` 是基址 + 变址 + 位移的形式，属于相对基址变址寻址。A 立即数寻址操作数直接在指令里；B 直接寻址是 `[3100h]`；C 基址变址寻址是 `[EBX+ESI]`，没有位移。

### C. PE与调试

12. B。`.idata` 是 import data，存放导入表信息。A `.text` 存放机器代码；C `.rsrc` 存放资源；D `.reloc` 存放重定位信息。

13. A。IAT 在运行时由 Loader 填入导入函数的真实地址。B 编译时函数地址未知；C 返回地址在栈帧中；D IAT 记录外部 DLL 函数地址。

14. B。VA = ImageBase + RVA = 0x00400000 + 0x00011000 = 0x00411000。A 计算错误；C 这是 RVA 本身；D 这是 ImageBase。

15. A。F7 是单步步入（遇到 CALL 进入子函数），F8 是单步步过（遇到 CALL 不进入）。B 顺序颠倒；C F9 是运行；D F9 是运行。

16. B。按 F5 可生成 C/C++ 风格伪代码。A F2 是设置断点；C F7 是单步步入；D F9 是运行。

### D. 漏洞类型与利用

17. B。`%n` 会把已输出字符数写入指定地址，可实现任意地址写入。A `%x` 读取栈上数据；C 整数溢出是计算错误；D 虚函数攻击依赖 vptr/vtable。

18. A。整数溢出本身是计算错误，但可能作为前置条件引发缓冲区溢出。B 整数溢出可发生在任何位宽；C 整数溢出可导致缓冲区溢出；D 整数溢出可影响内存分配。

19. A。攻击者通过篡改 vptr 或 vtable 来劫持虚函数调用。B 虚函数攻击也可通过栈溢出实现；C 虚函数调用依赖 vptr；D vtable 通常在代码段，不在栈上。

20. B。DEP 阻止数据区代码执行，ROP 通过复用已有可执行代码片段绕过。A DEP 会阻止栈上代码执行；C DEP 不能被攻击者关闭；D 整数溢出不能绕过 DEP。

21. A。ASLR 随机化关键地址布局，但可通过信息泄露绕过。B ASLR 不能阻止所有利用；C 程序逻辑不会被随机化；D ASLR 也保护堆和模块地址。

22. A。ROP 通过复用已有代码段中的 gadget 绕过 DEP。B ROP 执行的是已有代码，不是新注入代码；C ROP 也可用于 32 位；D ROP 依赖栈上的返回地址。

23. A。污点源是不可信输入，Sink 是危险操作点。B 污点分析可发现多种漏洞；C 污点分析可以是静态或动态；D Source 和 Sink 的定义颠倒了。

24. A。符号执行适合求解深路径约束，模糊测试适合快速探索大量输入。B 二者是不同技术；C 符号执行更精确；D 符号执行需要约束求解器。

### E. Web安全

25. A。XSS 让脚本在用户浏览器中执行，SQL 注入改变后端数据库查询语义。B XSS 不攻击数据库；C 二者有本质区别；D 存储式和反射式是 XSS 的类型。

26. A。存储式 XSS 风险通常更高，因为恶意脚本保存在服务器端，用户访问正常页面时触发。B 反射式需要用户点击链接；C 存储式风险更高；D 存储式可影响所有访问者。

27. A。参数化查询把 SQL 结构和用户输入分离，黑名单容易被编码、注释等方式绕过。B 速度不是主要优势；C 黑名单能防御部分攻击；D 参数化查询需要数据库支持。

28. A。LFI 包含本地文件，RFI 需要 allow_url_fopen 和 allow_url_include 开启。B 文件包含可包含多种文件类型；C 文件包含也可执行代码；D RFI 需要特定配置。

29. A。用户可控数据直接进入 unserialize()，触发魔术方法执行恶意操作。B PHP 支持序列化；C 反序列化可影响多种操作；D 魔术方法可执行多种操作。

30. A。CSRF 利用用户已登录身份发起请求，XSS 注入脚本在浏览器执行。B CSRF 不攻击数据库；C 二者有区别；D CSRF 也可通过 POST 实现。

## 三、简答题答案

41. 程序崩溃只是异常现象，不等于漏洞可利用。从三个角度说明：
- **可控性**：崩溃是否由攻击者可控的输入触发？如果只能随机崩溃，利用价值低。
- **覆盖目标**：崩溃位置是否能被精确控制？能否覆盖返回地址、函数指针等控制数据？
- **防护机制**：即使能覆盖，ASLR、DEP、Canary 等防护是否阻断利用链？

42. 栈溢出和堆溢出在攻击目标上的常见差异：
- **栈溢出**：常关注栈帧中的相邻局部变量、saved EBP、返回地址，目标是劫持控制流。
- **堆溢出**：常关注相邻堆块的对象字段、函数指针、虚表指针、堆管理元数据（块首、Flink/Blink），目标是破坏分配器逻辑或劫持间接调用。

43. Loader 加载 PE 文件的主要步骤：
1. 解析 DOS Header，找到 PE Header。
2. 读取 PE Signature、COFF Header、Optional Header。
3. 为 Image 分配虚拟地址空间。
4. 按 Section Table 映射各节到内存。
5. 查找 Import Table，加载所需 DLL，解析函数地址填入 IAT。
6. 如果 ASLR 生效或 ImageBase 被占用，进行 relocation 修正。
7. 处理 TLS 等初始化逻辑。
8. 跳转到入口点执行。

44. ROP 能绕过 DEP/NX 的原因：
- DEP 将栈/堆等数据区标记为不可执行，阻止直接执行注入的 shellcode。
- ROP 不执行新代码，而是复用程序或库中已有的可执行代码片段（gadget）。
- 每个 gadget 以 `ret` 结尾，通过栈上的返回地址链依次跳转。
- 因此 ROP 执行的是已有的可执行代码段，不受 DEP 限制。

45. 参数化查询比黑名单过滤更能防御 SQL 注入的原因：
- 参数化查询把 SQL 结构和用户输入分离，数据库按参数处理输入，输入不会被当作 SQL 语法解析。
- 黑名单过滤依赖枚举危险形式，容易被编码、注释、大小写、数据库函数和语法差异绕过。

46. 符号执行发现数组越界漏洞的思路：
1. 将输入变量符号化（如 x = X）。
2. 模拟程序执行，在每个条件分支记录路径约束。
3. 执行到数组访问时，根据符号表达式判断是否可能越界。
4. 将路径约束与越界条件组合，交给约束求解器求解。
5. 如果有解，说明存在输入能触发越界。

47. 存储式 XSS 和反射式 XSS 的区别：
- **存储式 XSS**：恶意脚本保存在服务器端（如数据库），用户访问正常业务页面时触发，风险更高，影响范围更广。
- **反射式 XSS**：恶意脚本通过 URL 参数反射，需要用户点击特定链接，风险相对较低。

48. 真实攻击中病毒、蠕虫、木马混合出现的原因：
- 真实攻击常分成"进入、控制、扩散、变现"几步。
- 三类恶意代码经常分工协作：蠕虫利用漏洞进入系统后投放木马用于长期控制，木马可能带有自传播模块表现出蠕虫特征，病毒感染宿主后也可能下载木马组件。
- 分类的目的是理解主要靠什么传播、靠什么控制、造成什么后果。

## 四、综合题答案

### 综合题 1：栈溢出攻击与防护绕过

49. 栈帧布局（从低地址到高地址）：

```
低地址
┌──────────────┐
│   buf[32]    │  ← 缓冲区
├──────────────┤
│ saved EBP[4] │  ← 旧帧指针
├──────────────┤
│ 返回地址[4]  │  ← 函数返回后跳转位置
├──────────────┤
│  函数参数    │
└──────────────┘
高地址
```

50. 需要输入 36 字节（32 字节 buf + 4 字节 saved EBP）才能开始覆盖返回地址。

51. 攻击思路：
- 构造 36 字节填充数据覆盖 buf 和 saved EBP。
- 接下来 4 字节填入 `system()` 函数的已知地址。
- 再后面 4 字节可以是返回地址（如 exit() 或任意地址）。
- 函数返回时，`ret` 会跳到 `system()`，执行系统命令。

**[待手动检验：需要确认 system() 的调用约定和参数传递方式]**

52. 如果开启了 Canary：
- Canary 位于 buf 和 saved EBP 之间。
- 函数返回前会检查 Canary 是否被破坏。
- 攻击者需要先泄露 Canary 值（通过任意读漏洞或其他方式）。
- 然后在溢出数据中填入正确的 Canary 值，绕过校验。
- 再继续覆盖返回地址。

### 综合题 2：PE注入与调试分析

53. RVA 计算：
- 注入代码 RVA = VA - ImageBase = 0x00405000 - 0x00400000 = 0x00005000
- 修改 PE 入口点时应填写 0x00005000

54. MessageBoxA 参数压栈顺序（从右到左）：
1. uType（消息框风格，通常为 0）
2. lpCaption（标题字符串地址）
3. lpText（内容字符串地址）
4. hWnd（窗口句柄，通常为 0/NULL）

55. 跳回原入口点的原因：
- 注入代码是在原程序执行前插入的额外逻辑。
- 执行完注入代码后，需要跳回原入口点，让原程序继续正常运行。
- 否则原程序的初始化和主逻辑不会执行。

56. OllyDbg 调试步骤：
- 打开程序后，EIP 停在入口点，可以通过 OllyDbg 标题栏或寄存器窗口查看 EIP 值。
- 入口点 VA = ImageBase + 原入口点 RVA = 0x00400000 + 0x00003000 = 0x0043000（原入口点）。
- 验证注入：运行程序，如果弹出 MessageBoxA 对话框，说明注入成功。

**[待手动检验：需要确认入口点 VA 的具体计算]**

### 综合题 3：Web攻击链串联

57. SQL 注入绕过登录：
- 在 user 参数中输入 `' OR 1=1 #`，使 SQL 查询变成 `SELECT * FROM user WHERE user='' OR 1=1 #' AND pwd='...'`。
- `OR 1=1` 使条件恒真，`#` 注释掉后续密码检查。
- 可绕过登录验证，无需知道正确密码。

58. 上传 + 文件包含串联：
- 上传一个包含 PHP 代码的图片文件（如 `shell.php.jpg` 或直接修改后缀）。
- 服务器仅检查 MIME 类型，攻击者可伪造 `Content-Type: image/jpeg`。
- 通过文件包含参数 `file=upload/shell.jpg` 访问上传的文件。
- PHP `include` 处理文件内容时，可能执行其中的 PHP 代码。

59. 防御措施：
- **SQL 注入**：使用参数化查询或预编译语句。
- **文件上传**：白名单后缀检查、真实类型校验、图片重编码、随机重命名、上传目录禁脚本执行。
- **文件包含**：白名单限制可包含的文件，禁止用户可控路径。

### 综合题 4：漏洞挖掘方法选择

60. 优先选择**模糊测试（Fuzzing）**：
- 模糊测试通过大量随机或变异输入快速触发崩溃。
- 适合快速发现缓冲区溢出、整数溢出等内存安全问题。
- 不需要源码，适合闭源程序。

61. 优先选择**污点分析**：
- 污点分析跟踪不可信输入（Source）是否传播到危险操作点（Sink）。
- 适合分析 SQL 注入、命令注入等由外部输入触发的问题。
- 可以是静态或动态分析。

62. 优先选择**符号执行**：
- 符号执行将输入符号化，记录路径约束。
- 适合求解深路径约束，找到能触发特定代码路径的具体输入。
- 可以精确分析越界访问的触发条件。

63. 应使用**程序切片**：
- 程序切片根据切片准则提取与目标变量或语句相关的代码。
- 后向切片追溯影响目标变量的前置语句。
- 前向切片分析目标变量影响的后续语句。
- 可有效缩小分析范围。
