# Lecture01 课程概要和基本概念 - 习题答案

## A. 基础查表题答案

1. A。病毒强调依附宿主，蠕虫强调网络扩散，木马强调伪装和控制。B、C、D 选项的顺序都与手册描述不符。

2. B。CVE（Common Vulnerabilities and Exposures）是给具体漏洞实例统一编号的体系，像身份证号。A 是 CWE 的职责（弱点类型分类）；C 是 CVSS 的职责（评分体系）；D 描述的是受影响版本清单，是 NVD 条目中的一个字段。

3. B。补丁刚公开、系统尚未普遍更新时，按 1day 风险理解。A 是 0day，指漏洞尚未公开、厂商通常不知道或没有补丁；C 描述的是已修复状态，风险已降低；D 是弱点分类概念，不是时间线阶段。

4. A。肉鸡就是被攻击者控制的主机。手册指出肉鸡通常依赖"软件漏洞 + 木马"形成。B 跳板机强调"中转隐藏身份"的功能；C、D 是设备类型，不是控制状态。

5. C。黑箱测试从外部视角开始，内部信息最少。A 白箱测试需要源码和架构信息；B 隐秘测试强调被测单位知情范围很小；D 灰箱测试介于黑箱和白箱之间，测试者掌握部分内部信息。

6. C。CVSS（Common Vulnerability Scoring System）是漏洞评分系统。A CVE 是漏洞编号；B CWE 是弱点类型分类；D NVD 是漏洞数据库。

7. 对。能被外部输入稳定触发并造成安全后果时，已经接近安全漏洞的定义。手册明确指出：只有当缺陷能被外部输入触发，并导致越权、信息泄露、代码执行等安全后果时，才更接近漏洞。

8. 对。手册强调：病毒、蠕虫、木马不是互斥标签。真实攻击中经常混合，例如蠕虫进入系统后投放木马用于长期控制。

9. 对。白箱测试需要源码、拓扑、账号信息等，覆盖更深，效率更高。

10. 对。1day 的风险在于：攻击者可以通过比较补丁前后代码差异，推测漏洞位置，快速构造 exploit。很多真实攻击不是靠 0day，而是靠大家没及时修补的 1day。

11. 错。手册明确指出：不是所有 Bug 都是漏洞。按钮颜色显示错误、普通页面排版错乱、某个本地工具偶尔崩溃，通常只是功能或稳定性问题，不属于安全漏洞。

12. 错。手册指出：渗透测试的核心不是工具，而是授权范围内的风险验证。没有授权的扫描、利用、入侵，即使技术动作类似，也不是渗透测试。

13. `Common Vulnerabilities and Exposures`。

14. `National Vulnerability Database`。

15. `Penetration Testing`。

16. `Common Vulnerability Scoring System`。

## B. 理解应用题答案

17. B。浏览器、插件、文档解析器本身就是程序，恶意内容可以利用解析漏洞触发执行。A 错误：URL 本身不是可执行文件；C 错误：操作系统不会因为点击链接就自动下载运行木马；D 错误：网页不会自动安装插件，插件执行也受沙箱限制。

18. D。漏洞库的核心价值是统一编号、分类、评分和参考信息，让厂商、安全人员、扫描器能说同一种语言。A 错误：漏洞库不存储恶意代码样本；B 错误：这是威胁情报或取证系统的职责；C 错误：漏洞库提供信息参考，不直接执行补丁修复。

19. A。渗透测试是在授权范围内模拟攻击者方法做风险验证。B 错误：未取得授权的测试不是渗透测试，而是攻击行为；C 错误：扫描结果只是渗透测试的一部分；D 错误：渗透测试强调在授权范围内验证风险，不是走完整攻击链。

20. A。0day 的危险在于信息不对称——防御者不知道漏洞存在，没有补丁；1day 的危险在于补丁已公开但大量系统未修补，攻击者可逆向补丁快速构造利用。B 错误：1day 并不意味着漏洞已修复；C 错误：0day 和 1day 的区别不在于影响范围或评分；D 错误：二者描述的是漏洞公开状态和修复状态，不是利用时间。

21. Bug 是程序缺陷，漏洞强调可被利用并造成安全影响。能崩溃不等于能利用，能利用才更接近漏洞。手册用公式表达：软件错误/缺陷 >= Bug，Bug 中可被攻击者利用并造成安全影响的部分 = 漏洞。

22. 因为漏洞库会整理编号、类型、影响版本、评分和参考链接，既方便研究，也方便排查资产、安排修复和写报告。漏洞库的价值不是"背编号"，而是帮助防御者定位资产、判断风险、安排补丁、写报告、配置扫描器。

23. 因为真实攻击常分成"进入、控制、扩散、变现"几步，三类恶意代码经常分工协作。例如蠕虫利用漏洞进入系统后投放木马，木马也可能带有自传播模块表现出蠕虫特征，病毒感染宿主后也可能下载木马组件。

## C. 综合融会题答案

24. 伪装成正常工具并诱导用户运行，这部分更接近木马。手册指出木马的关键词是伪装、诱导、隐蔽控制。

25. 扫描局域网、尝试向其他主机扩散，这部分更接近蠕虫。手册指出蠕虫强调网络独立传播。

26. 因为它同时具备伪装控制、远程操作和横向扩散等多个特征，单一标签只能说明主特征，不能完整概括行为链。手册强调：分类的目的不是给每个恶意程序贴唯一名字，而是理解它主要靠什么传播、靠什么控制、造成什么后果。

27. 因为补丁公开后，攻击者可以围绕补丁和公开分析推断漏洞位置，而大量系统尚未更新时仍然暴露在可利用窗口里。手册指出：1day 的危险在于补丁已经公开但修复未必跟上。

28. 尽快完成资产排查、验证受影响版本并安排补丁或缓解措施。

29. 因为补丁和分析文章会帮助攻击者逆向漏洞点，缩短利用开发时间。手册指出：攻击者可以通过比较补丁前后代码差异，推测漏洞位置，写出 exploit。

30. 更接近黑箱测试。手册指出：黑箱测试从公开信息、端口、页面、接口开始。

31. 会更接近白箱测试。手册指出：白箱测试能获得源码、拓扑、账号、架构等信息。

32. 隐秘测试比普通黑箱测试更强调"测试存在本身尽量不被发现"，重点还包括检测和响应能力。手册指出：隐秘测试的重点不只是漏洞，也包括检测响应。
