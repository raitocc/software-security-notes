## A. 基础查表题

这一部分主要考格式化字符串、整数溢出、虚函数攻击、注入和越权的基础概念。

### 选择题

1. 下列哪种写法更容易引发格式化字符串漏洞？

A. `printf(user_input);`  
B. `printf("%s", user_input);`  
C. `puts(user_input);`  
D. `fgets(buf, 200, stdin);`

2. `%n` 在格式化字符串中的作用更接近：

A. 把已输出字符数写入对应地址  
B. 按十六进制形式输出栈上数据  
C. 把参数当地址读取字符串内容  
D. 按指针形式输出一个地址值

3. `AAAA%x%x%x%s` 中，若前面的读取使 `0x41414141` 成为 `%s` 对应参数，`%s` 更可能做什么？

A. 从地址 `0x41414141` 读取字符串  
B. 输出整数数值 `0x41414141`  
C. 把 `AAAA` 作为格式宽度处理  
D. 将四个 `A` 转换为一个栈帧

4. 整数溢出和缓冲区溢出的关系更接近：

A. 整数溢出常导致错误长度判断或错误分配，再诱发缓冲区溢出  
B. 整数溢出会直接覆盖返回地址，而不需要后续内存操作  
C. 缓冲区溢出主要用于修复整数计算中的符号问题  
D. 二者都主要发生在格式化输出的参数解析阶段

5. C++ 虚函数攻击中，vptr 通常保存在哪里？

A. 对象的内存空间中  
B. 类声明文本中  
C. 操作系统页表中  
D. PE 文件 DOS 头中

6. 水平越权更接近下列哪种场景？

A. 用户 A 修改订单编号后访问用户 B 的订单  
B. 普通用户直接访问管理员配置页面  
C. 攻击者通过 SQL 拼接改变查询条件  
D. 程序把用户输入当作系统命令执行

### 判断题

7. 格式化字符串漏洞的关键是格式串被攻击者控制。

8. `%x` 和 `%p` 常可用于泄露地址或栈上数据，进而辅助绕过地址随机化。

9. 整数溢出本身通常是数值计算错误，但它可能成为后续内存破坏的触发条件。

10. SQL 注入、命令注入、SOAP 注入都可以从“数据进入解释器语法”这一角度理解。

## B. 理解应用题

这一部分要求你解释漏洞形成链条，而不是只记住符号。

### 选择题

15. 为什么 `printf("%s", user_input)` 比 `printf(user_input)` 更安全？

A. 前者由程序固定格式串，用户输入只作为普通字符串参数  
B. 前者会把用户输入自动复制到内核态缓冲区  
C. 前者会让 `%n` 按普通字符串内容显示而不参与参数解析  
D. 前者会按栈帧大小自动限制用户输入的最大长度

16. `printf("%100d%n", num, &num)` 中，`%100d` 对 `%n` 的影响更接近：

A. 增加到 `%n` 前已经输出的字符数  
B. 改变 `num` 的整数类型和符号位  
C. 让 `%n` 改为读取字符串而非写入  
D. 改变 `&num` 在参数列表中的相对位置

17. 关于 `short all_len = len1 + len2; if (all_len < MAX_INFO)`，下列理解更合理的是：

A. `len1 + len2` 超过 short 范围后可能变成负数，从而绕过检查  
B. `short` 会自动扩展范围以保存两个 30000 的加法结果  
C. `if` 条件会根据字符串真实长度重新计算 `all_len`  
D. `strcpy` 和 `strcat` 会根据 `MAX_INFO` 自动截断输入

18. 关于虚函数攻击，下列说法更贴近课程内容的是：

A. 攻击者可篡改 vptr 或虚表项，使虚函数调用跳到目标地址  
B. 虚函数调用主要依赖编译期函数名，不通过对象内存中的指针间接定位  
C. vtable 主要保存对象的普通字段和字符串内容  
D. 覆盖 vptr 后主要影响对象字段布局，不影响间接调用目标

19. SOAP 注入和 SQL 注入的共同点更接近：

A. 外部输入进入解释性结构，改变原本语义  
B. 二者都通过覆盖返回地址执行 shellcode  
C. 二者都依赖 C++ 虚表指针被覆盖  
D. 二者都主要通过整数长度回绕绕过内存分配

## C. 综合融会题

这一部分更接近考试中的代码推导和场景分析。

### 综合题 1：格式化字符串读取

某程序代码如下：

```c
char str[200];
fgets(str, 200, stdin);
printf(str);
```

输入 `AAAA%x%x%x%x` 后，输出末尾出现 `41414141`。

24. `41414141` 为什么和 `AAAA` 有关？
25. 如果把最后一个 `%x` 改成 `%s`，程序会如何解释 `41414141`？
26. 这种读取能力为什么可能帮助绕过 ASLR？

### 综合题 2：格式化字符串写入

某程序中存在：

```c
char buffer[100];
sprintf(buffer, argv[1]);
```

命令行参数为：

```text
aaaabbbbcc%n
```

27. 到 `%n` 之前已经输出了多少个字符？
28. 为什么课件中会把 `aaaa` 解释成地址 `0x61616161`？
29. 如果想影响 `%n` 写入的数值，可以利用格式字符串的什么特性？

### 综合题 3：整数溢出链条

某函数逻辑如下：

```c
unsigned int size = len + 1;
char *buffer = malloc(size);
memcpy(buffer, data, len);
```

攻击者传入 `len = 0xFFFFFFFF`。

30. `size` 可能变成多少？
31. 为什么 `malloc` 和 `memcpy` 之间会产生危险差异？
32. 这个漏洞链条最终更可能表现成哪类内存破坏？

### 综合题 4：越权与注入

某 Web 系统中，普通用户访问 `/admin/delete?id=10` 可以删除用户；另一个接口 `/order?id=1002` 可通过修改订单号查看他人订单。

33. 前者更接近哪类越权？
34. 后者更接近哪类越权？
35. 两者共同的服务端根因是什么？
