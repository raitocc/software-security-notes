## 二、判断题（10题）

31. 所有软件 Bug 都属于安全漏洞，因为它们都会影响系统稳定性。

32. 渗透测试的核心是使用先进的扫描工具，工具越强大测试效果越好。

33. 栈帧里通常会有返回地址、保存的旧 EBP 和局部变量，全局变量不属于单次函数调用栈帧的核心内容。

34. PE 文件中可以包含图标、菜单、位图等资源信息。

35. 用户态调试器中看到的程序地址通常是虚拟地址，不是物理地址。

36. 缓冲区溢出只能发生在栈区，堆区不存在溢出风险。

37. 格式化字符串漏洞中，使用固定格式串（如 `printf("%s", user_input)`）可以避免漏洞。

38. ASLR 是利用缓解机制，不能从根本上修复漏洞本身。

39. POST 请求的数据放在请求体中，因此 POST 请求的数据不会被泄露。

40. HttpOnly 可以降低脚本直接读取 Cookie 的风险，但脚本仍可能借当前登录态发起请求。

## 四、综合题（4题）

### 综合题 1：栈溢出攻击与防护绕过

某 32 位 Windows 程序存在如下函数：

```c
void vulnerable(char *input) {
    char buf[32];
    strcpy(buf, input);
}
```

程序编译时开启了 ASLR，但未开启 DEP 和 Canary。攻击者已知该程序加载的某个 DLL 中存在 `system()` 函数，且该 DLL 的基址在同一进程中固定不变（ASLR 未对该 DLL 生效）。

49. 画出该函数栈帧的典型布局（从低地址到高地址），标出 buf、saved EBP、返回地址的相对位置。
50. 攻击者需要输入多少字节才能开始覆盖返回地址？
51. 攻击者如何利用已知的 `system()` 函数地址构造攻击 payload？请简述思路。
52. 如果程序同时开启了 Canary，攻击者的利用思路会有什么变化？

### 综合题 2：PE注入与调试分析

某攻击者拿到一个 Windows 可执行程序 `target.exe`，ImageBase = 0x00400000，原入口点 RVA = 0x00003000。攻击者计划在空白代码区注入一段调用 `MessageBoxA` 的代码，注入代码起始 VA = 0x00405000。

53. 注入代码的 RVA 是多少？修改 PE 入口点时应填写什么值？
54. 注入代码中调用 `MessageBoxA` 的参数压栈顺序是什么？
55. 注入代码执行完后，为什么要跳回原入口点？
56. 如果使用 OllyDbg 调试该程序，如何定位入口点？如何验证注入是否成功？

### 综合题 3：Web攻击链串联

某 Web 应用存在以下功能：
- 用户登录页：`POST /login.php`，参数 `user` 和 `pwd` 直接拼接进 SQL 查询
- 头像上传：`POST /upload.php`，仅检查 MIME 类型是否为 `image/jpeg`
- 文件包含：`GET /page.php?file=xxx`

57. 攻击者如何利用 SQL 注入绕过登录验证？
58. 攻击者如何利用上传功能和文件包含漏洞串联成代码执行链？
59. 从防御角度，针对上述三个漏洞分别应采取什么措施？

### 综合题 4：漏洞挖掘方法选择

60. 如果要快速发现程序是否存在崩溃类漏洞，应优先选择哪种挖掘方法？为什么？
61. 如果要分析程序是否受到外部输入的影响（如 SQL 注入、命令注入），应优先选择哪种方法？为什么？
