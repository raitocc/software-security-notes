## 一、选择题（30题）

1. 下列关于 CVE 和 CWE 的说法，更合理的是：

A. CVE 和 CWE 都是由国际标准组织制定的漏洞评分与定级系统
B. CWE 给具体漏洞实例编号，CVE 给漏洞类型分类
C. CVE 和 CWE 主要作为企业内部安全审计的参考代码库
D. CVE 给具体漏洞实例编号，CWE 给漏洞类型分类

2. 某漏洞补丁刚发布一周，安全社区已有公开分析文章，但大量服务器尚未更新。这个阶段更接近：

A. 漏洞尚未被官方承认的评估定级期
B. Nday 且已进入普遍修复的安全静默期
C. 0day 漏洞的高危潜伏期
D. 1day 漏洞的实战利用窗口期

3. 下列关于肉鸡（Bot）的说法，更合理的是：

A. 个人主机的网络架构天然排斥被作为肉鸡，主要威胁集中在数据中心服务器端
B. 一旦主机被纳入僵尸网络，常规的防病毒软件机制将完全失效
C. 肉鸡的形成高度依赖于网络蠕虫的自主传播与复制机制
D. 肉鸡通常依赖"软件漏洞 + 木马"的组合形式完成控制权窃取

4. 关于渗透测试和漏洞扫描，下列说法更合理的是：

A. 渗透测试是漏洞扫描在复杂网络环境下的底层进阶替代方案
B. 漏洞扫描输出的报告在合规性审查中等效于渗透测试的专家验证结果
C. 渗透测试是授权范围内的风险验证，漏洞扫描侧重自动化检测
D. 渗透测试往往在未告知系统所有者的前提下进行，以追求测试的真实性

5. 关于黑箱测试和隐秘测试，下列说法更合理的是：

A. 黑箱测试依赖于对目标系统架构的深度解析，而隐秘测试注重广度探测
B. 隐秘测试旨在发掘系统逻辑深处的漏洞，通常不需要评估防御团队的应急响应能力
C. 两者在测试方法论上同属于代码白盒测试的延伸分支应用
D. 黑箱测试从外部视角开始，隐秘测试强调测试行为本身不被目标防御机制察觉

6. 某 32 位程序函数栈帧中，`buf[12]` 与返回地址之间有 4 字节 saved EBP。若程序关闭了各项保护，通过 `strcpy(buf, input)` 输入 20 字节，下列说法更合理的是：

A. 缓冲区溢出会按照变量声明的逆序依次覆盖栈帧中的保存指针
B. 覆盖操作将局限于局部变量分配的内存范围内，不会波及控制流元数据
C. 该操作会破坏栈底指针，但通常不足以触及函数调用链的关键返回地址
D. 覆盖了 buf 的 12 字节、saved EBP 的 4 字节以及返回地址的低 4 字节

7. 关于堆管理器中 `malloc` 返回的指针，下列说法更合理的是：

A. 指向堆分配表中的对应条目，便于系统在后续操作中快速索引空闲块
B. 指向内存块的首部结构，使得应用程序可以直接读取块大小等状态信息
C. 指向分配内存块的用户数据区起始位置，但不包含块首管理元数据
D. 指向操作系统分配给该进程的物理内存页起始地址，以优化缓存命中率

8. 下列关于栈和堆的安全问题，更合理的是：

A. 栈溢出的破坏力往往局限于函数内部的局部变量，而堆溢出更容易劫持系统级执行流
B. 堆的动态分配特性使其在现代操作系统中拥有固定的绝对基址，相对栈结构更易于利用
C. 栈溢出常直接威胁控制流返回地址，堆溢出多通过覆盖堆元数据或相邻对象指针实现利用
D. 栈与堆在内存破坏机制上共享相同的利用逻辑，均通过修改异常处理链表完成劫持

9. 关于函数调用流程，下列说法更合理的是：

A. `call` 指令负责压入返回地址并跳转，被调用函数建立自身栈帧，`ret` 从栈中取出地址并跳回
B. 函数调用的返回地址由编译器在预处理阶段直接硬编码在可执行文件的重定位表中
C. `call` 指令在跳转前会自动分配目标函数所需的局部变量栈空间，`ret` 则负责清理
D. 返回地址由被调用函数在执行完毕后，手动向指令寄存器 EIP 写入计算后的偏移值

13. 关于 PE 文件中的 IAT（Import Address Table），下列说法更合理的是：

B. 编译器在生成可执行文件时，会根据系统环境变量提前在 IAT 中固化模块的物理基址
C. IAT 在程序被操作系统加载至内存时，由 Loader 解析并填入相关导入函数的真实内存地址

17. 在格式化字符串漏洞中，格式符 `%n` 引发的安全威胁机制主要表现为：

A. 将当前已打印的字符总数写入到由栈上指针指向的特定内存地址中
B. 篡改对象的虚函数表结构，使得后续的多态调用偏离预期的控制流
C. 引发目标变量的数值溢出，进而使得随后的缓冲区拷贝逻辑绕过长度校验
D. 从堆栈深处越界读取包含敏感凭证的连续内存数据

18. 关于整数溢出（Integer Overflow）漏洞，下列说法更合理的是：

A. 现代 64 位操作系统的寄存器位宽天然屏蔽了此类算术异常导致的系统级内存破坏
B. 发生溢出后，内存分配函数通常会主动捕获符号位异常并抛出分段错误中断程序
C. 整数溢出本质上属于算术运算异常，但常作为前置触发条件引发随后的缓冲区越界读写
D. 整数溢出与缓冲区溢出属于平行的漏洞分类，两者在实际利用链中往往互斥存在

19. 关于 C++ 程序的虚函数攻击，下列分析更合理的是：

A. 虚函数攻击的实施深度依赖于对底层堆块管理的操控，尤其需要触发特定的双向链表解链宏
B. 对象的虚表元数据驻留于当前调用栈帧中，使得常规的局部变量越界即可破坏其完整性
C. 虚函数的跳转表通常被编译器硬编码在可变数据段，攻击者主要通过栈溢出来篡改映射关联
D. 攻击者常通过覆盖对象的虚表指针（vptr）或伪造虚函数表（vtable）来劫持多态调用执行流

20. 某程序存在典型的栈溢出漏洞，但其运行环境开启了 DEP/NX（数据执行保护）。面对此种防御，攻击者更倾向于采用的利用思路是：

A. 将恶意 Shellcode 放置于堆区等动态分配的内存块中以规避栈执行权限的安全限制
B. 利用 ROP（返回导向编程）技术，将内存中现有的合法代码片段拼接组合执行
C. 通过精巧的格式化字符串读写原语直接修改段寄存器的标志位，在运行时动态卸载 DEP
D. 结合堆机制的释放后重用缺陷，将控制流直接导向系统预设的错误处理句柄以执行命令

21. 关于 ASLR（地址空间布局随机化）机制，下列评估更合理的是：

A. ASLR 通过每次运行加载不同的基址增加利用门槛，但常被结合信息泄露漏洞予以绕过
B. 该安全机制重点针对栈空间和堆空间进行位置随机化，一般不对操作系统的核心动态库实施偏移
C. ASLR 能够在内存分页管理层面拦截对非预期区段的越界写入，从底层阻断缓冲区溢出的发生
D. 开启 ASLR 后，程序在内存中的机器指令流会被加密混淆，从而大幅延缓逆向分析的进度

22. 关于 ROP（Return-Oriented Programming）利用技术，下列原理解析更合理的是：

A. ROP 的核心在于复用进程内存空间中已具备执行权限的短小指令片段（Gadgets），规避 DEP 限制
B. 其利用过程高度依赖对堆块管理结构中 Chunk 头部信息的覆盖，使得释放操作触发恶意指针解引用
C. ROP 主要通过向目标进程反射注入外部动态链接库，并在新开辟的线程上下文中执行恶意逻辑
D. 该技术要求攻击者将定制化编写的汇编指令序列注入程序的栈帧空间，并篡改 EIP 直接跳转执行

23. 在软件安全静态分析的污点分析（Taint Analysis）中，核心概念的界定更倾向于：

A. 该技术侧重于在编译阶段，通过代码语义检查将可能导致越界访问的高危 C 库函数调用予以剥离或替换
B. 汇聚点通常特指业务数据的最初始加载函数，而污点源则是在程序流程终止阶段执行的核心 API 接口
C. 污点分析依赖于向目标服务高频注入大批量的畸形结构数据，借由监控系统错误日志来反推漏洞触发路径
D. 污点源（Source）代表不可信的外部输入注入点，汇聚点（Sink）代表触发潜在安全风险的敏感操作执行点

24. 关于符号执行（Symbolic Execution）与模糊测试（Fuzzing），两者的典型技术特征差异表现在：

A. 符号执行在分析海量代码的现代商业软件时，得益于不运行真实代码的特性，整体耗时通常远短于依赖随机变异的模糊测试
B. 两者在漏洞发掘的方法论上均属于典型的静态代码审计分支，不依赖于目标程序的实际运行环境即可输出详细的安全报告
C. 符号执行擅长通过严谨的数学约束求解深层逻辑分支的特定触发条件，而模糊测试更适合在大规模状态空间中快速探测崩溃
D. 模糊测试在发掘具有复杂多重条件校验的深层逻辑漏洞时，其探测精度往往显著优于基于路径公式求解的符号执行引擎

25. 跨站脚本攻击（XSS）与 SQL 注入在攻击意图与目标维度上的主要差异体现为：

A. 前者利用了后端应用对输入数据的对象反序列化缺陷，后者利用了前端页面在构建 DOM 树时的安全过滤策略失效
B. XSS 的恶意脚本执行于受害者的浏览器沙箱中，而 SQL 注入意图篡改或窃取后端数据库服务器的核心业务数据
C. XSS 主要针对后端 Web 容器的系统层级权限展开渗透，SQL 注入则侧重于诱骗前端用户提交保存在本地的加密凭证
D. XSS 漏洞的终极攻击目标是以获取数据库服务器所在网段的控制权为标志，SQL 注入则局限于对当前用户在线会话的劫持

26. 探讨存储式 XSS 和反射式 XSS 的威胁模型，下列观点更合理的是：

A. 反射式 XSS 的恶意代码往往被记录在服务器的系统日志缓存中等待系统管理员查看，无需攻击者主动发送带有 Payload 的钓鱼链接
B. 两类跨站攻击在恶意脚本的载入机制上呈现相似的触发链条，其实际危害评级主要由目标网站的后端架构复杂度决定，而非漏洞分类本身
C. 存储式 XSS 的 Payload 被持久化写入服务器端，当其他合法用户正常浏览受污染页面时即被动触发，潜在感染波及面更广
D. 由于存储式 XSS 依赖 HTTP GET 请求中的 URL 栏明文参数传递，其攻击特征比反射式 XSS 更容易被网络边界的 Web 防火墙识别拦截

27. 在防范 Web 系统 SQL 注入风险时，业界通常推崇参数化查询（预编译）胜过黑名单字符过滤，主要由于：

A. 参数化查询机制强制依赖于前端的 JavaScript 数据加密组件，使得后端数据库处理逻辑能够直接规避来自非授权接口的明文攻击
B. 传统的黑名单过滤在面对高并发且含有大量复杂关联表的数据库请求时，会显著增加网络服务器的 CPU 负担并引发内存耗尽
C. 数据库中间件在处理参数化语句时，能够启动启发式扫描引擎自动识别并剔除输入数据中的恶意脚本结构，提升了查询的响应效率
D. 参数化查询从数据库引擎底层将 SQL 语句的指令语法与外部传入的数据载荷严格隔离，避免了转义逻辑遗漏导致的深层绕过风险

28. 关于 Web 系统中的文件包含漏洞（File Inclusion），下列特征分析更合理的是：

A. 无论 LFI 还是 RFI，其攻击的终极目的由于受限于后端代码逻辑，往往只能包含特定 `.php` 后缀的文件，难以对诸如纯文本或系统配置信息进行提取
B. RFI 漏洞的触发和执行机制天然能够免疫 Web 服务器在目录层级的读写权限约束，因此在常规默认配置的现代 Web 容器中依然能够保持极高的漏洞复现率
C. 本地文件包含（LFI）允许攻击者读取服务器操作系统的本地物理文件，而远程文件包含（RFI）通常需要目标 PHP 环境开启特定的网络流读取配置支持
D. 该类漏洞的危害上限仅仅停留在服务器内部敏感文件的越权读取泄露层面，攻击者往往难以借此途径在目标系统上进一步获取执行任意系统代码的权限

29. 导致 PHP 反序列化漏洞最终能够实现任意系统代码执行的核心诱因通常是：

A. 魔术方法在特定的对象实例化场景下被隐式调用的过程中，缺乏对函数指针类型的安全校验，导致攻击者能够直接覆写底层动态库的加载基址
B. 序列化格式的解析器引擎在处理超长编码字符串时发生堆栈内存溢出，攻击者借此利用提前布置的 ROP 链劫持了 PHP 守护进程的控制权
C. 用户可控的恶意序列化对象数据被传入 `unserialize()` 后，在对象重建或销毁的生命周期内自动触发了魔术方法内预埋的危险业务调用链
D. 反序列化操作在底层处理逻辑中会自动对传入的所有外部字符串实施强制的变量宏替换，使得原本作为字符串保存的数据被解析器直接投入代码执行

30. 跨站请求伪造（CSRF）与跨站脚本攻击（XSS）在实战攻击逻辑上的显著差异在于：

A. CSRF 利用受害者浏览器中已保存的合法登录凭证静默发起越权业务请求；而 XSS 则旨在直接将恶意脚本实体注入并运行于受害者的当前浏览器上下文中
B. CSRF 的战术核心在于通过跨域请求绕过后端防御以窃取数据库的极密业务信息；XSS 则更侧重于向目标 Web 应用发起大量异步请求导致后端的拒绝服务
C. 前者攻击链的成功实施必然要求受害者在当前页面主动点击精心伪装的重定向按键，后者的触发则完全依赖于后端服务在处理请求时未进行过滤的被动响应
D. CSRF 攻击主要通过网络中间人手段拦截并恶意篡改传输链路中的 HTTP 响应体数据实现，XSS 则是利用伪造的 HTTP Referer 头部绕过应用的前置身份验证

## 二、判断题（10题）

31. 在软件工程实践中，系统出现的任何业务逻辑上的报错或异常终止，都应被直接判定为安全漏洞，因为它们在特定极端负载下必定会对系统的整体可用性造成毁灭性破坏。
32. 开展企业级渗透测试工作时，决定测试深度的核心要素在于团队是否采购并部署了最高版本的商业自动化扫描套件，因为高级扫描策略通常具备取代人工对复杂业务逻辑进行全面安全审计的能力。
33. 在单次函数调用的底层栈帧内存模型中，全局声明的静态数据结构也会伴随调用过程被统一压入当前栈空间内，以保障多线程并发调用时各个函数局部状态的绝对隔离。
34. Windows 操作系统下的 PE（Portable Executable）可执行文件结构不仅包含用于运行的机器码和常量数据，还可以利用 `.rsrc` 等独立节区来封装程序展现所需的图标、对话框以及多媒体位图等丰富的资源信息。
35. 安全人员在使用用户态动态调试器（如 x64dbg 或 OllyDbg）逆向分析应用程序时，在内存监控窗口和反汇编面板中所观察到的寻址数值，实际上是操作系统底层映射的虚拟内存地址，而非硬件物理颗粒上的绝对访问地址。
36. 在现代操作系统的内存访问保护架构中，由于栈结构的后进先出特性，缓冲区溢出的安全风险仅仅局限于栈内存区；而由系统 API 动态分配的堆区由于受到更严密的分配器块头隔离保护，在逻辑上阻绝了发生越界覆写的可能性。
37. 在 C 语言的字符串处理场景中，若开发者严格采用固定格式化控制符（例如 `printf("%s", user_input)`）来替代将变量作为参数直接抛出（如 `printf(user_input)`），能够从语法层面极大降低触发格式化字符串漏洞的风险。
38. ASLR（地址空间布局随机化）作为一种主流的内存安全缓解机制，它通过大幅提高攻击者预测关键内存地址的计算难度来增强系统安全性，但其实质并没有在源码逻辑层面修复那些引发内存破坏的原始缺陷漏洞。
39. 由于 HTTP POST 请求协议在设计上将提交数据隐藏于不可见的请求体内而非暴露在 URL 参数中，因此应用开发者使用 POST 方法传输诸如密码等敏感用户凭证时，即可天然免疫在不安全网络传输链路中被旁路嗅探泄露的风险。
40. 将 Web 会话 Cookie 的 `HttpOnly` 属性配置为激活状态，能够极其有效地防范 JavaScript 恶意代码通过 `document.cookie` 接口实施的直接凭证窃取，但此举依然无法阻止浏览器在受到请求欺骗时自动在 HTTP 头部附带该凭证向同源后台发起交互。

## 三、简答题（8题）

**41. 漏洞评估与可利用性分析**
【场景背景】
安全工程师使用自动化 Fuzzer 对某开源图像处理库进行模糊测试时，监控系统报告程序在处理一个特定畸形图片时引发了 Access Violation（访问违例）并导致崩溃退出。工程师随后将该崩溃记录为潜在的内存破坏漏洞。
【问题】
仅仅发现程序崩溃并不意味着能够最终实现 RCE（远程代码执行）。在后续的漏洞定级和利用开发中，请从“导致崩溃的异常输入是否具备**可控性**”以及“崩溃引发的内存破坏是否能够**覆盖关键控制流目标**（如返回地址或函数指针）”这两个评估维度，简述为什么该崩溃可能“无法被利用”。

**42. 内存破坏的攻击层面差异：栈溢出与堆溢出**
【场景背景】
某 C 语言编写的网络服务被曝出存在两处安全隐患：一处是由于 `strcpy` 导致局部变量越界，引发了传统的栈溢出；另一处则是向 `malloc` 分配的动态内存中过度拷贝，导致了堆溢出。
【问题】
这两种溢出漏洞虽然成因相似，但在后续的利用手法和劫持目标上存在显著区别。请简述栈溢出通常直接威胁的关键栈帧结构是什么？而堆溢出往往通过破坏堆内存中相邻的哪些“数据结构”（如业务对象的虚表、或分配器的元数据）来间接达到劫持程序的最终目的？

**43. 可执行文件的底层机制：PE 加载与初始化**
【场景背景】
在分析一款疑似捆绑了恶意代码的 Windows 可执行文件（PE 文件）时，逆向分析师注意到该文件被双击运行后，操作系统并未直接将其全部文件内容按顺序映射到内存，而是交由底层的 PE Loader（加载器）执行了一系列复杂的装载操作。
【问题】
在控制权最终移交给程序的入口点（OEP）之前，PE Loader 至少需要完成哪些关键的初始化准备工作？（请围绕为各节区分配虚拟内存、以及处理导入地址表（IAT）这两个核心步骤进行简要阐述）。

**44. 缓解机制的极限对抗：ROP 技术的核心原理**
【场景背景】
某次安全实战中，攻击者在一个启用了 DEP/NX（数据执行保护）的 Linux 进程中成功利用了栈溢出漏洞。由于 DEP 强制剥夺了栈内存区域的“可执行”权限，攻击者发现直接跳往栈上的传统 Shellcode 会立刻触发系统的执行保护异常。
【问题】
为了在不关闭 DEP 保护的前提下完成任意命令执行，攻击者采用了 ROP（Return-Oriented Programming）技术。请简要说明该技术是如何通过复用程序原本所在内存（如 `.text` 段或系统动态库）中现存的合法指令片段，并通过精心布置栈数据将其像“积木”一样拼接，从而实现原本无法完成的恶意功能的？

**45. SQL 注入的终极防御：预编译的架构优势**
【场景背景】
某金融类 Web 应用的登录接口原本使用正则表达式黑名单过滤单引号（`'`）和 `SELECT` 等关键字来防御 SQL 注入，但由于过滤规则遗漏，被攻击者利用多字节编码技巧成功绕过并拖库。随后，开发团队彻底放弃了黑名单，全面采用了**参数化查询（预编译语句）**进行重构。
【问题】
从底层执行架构的角度来看，为什么说参数化查询比字符黑名单替换更能彻底防御 SQL 注入？（提示：请重点阐述在预编译机制下，数据库引擎是如何严格分离“SQL 语法结构解析”与“外部输入数据绑定”的过程的）。

**46. 自动化挖掘前沿：符号执行的应用逻辑**
【场景背景】
安全分析工具在审计某个包含复杂协议解析分支的代码时，发现了一个可能由于索引计算错误引发的深层数组越界漏洞。由于触发该路径需要输入数据同时满足几十个复杂的 `if-else` 数学校验条件，传统的黑盒 Fuzzing 几乎无法在短时间内随机碰撞出符合条件的特定输入。
【问题】
为了突破这一瓶颈，现代安全审计引入了符号执行（Symbolic Execution）技术。请简述该技术是如何将程序中的执行路径提取为“数学公式（约束条件）”，并利用求解器自动计算出能够精确引发上述数组越界特定代码分支的测试用例数据的？

**47. 跨站脚本的威胁演变：存储式与反射式的实战差异**
【场景背景】
某电商平台被安全团队报告存在两处 XSS 漏洞。漏洞 A 是存在于搜索框的反射式 XSS；漏洞 B 则是存在于商品评价留言板的存储式 XSS。
【问题】
这两类漏洞虽然都导致了恶意 JavaScript 脚本在受害者的浏览器中执行，但在真实的攻击传播链路和影响规模上差异巨大。请分别说明攻击者如何利用这两处漏洞触发攻击（如是否需要诱导点击链接？），并解释为何漏洞 B（存储式）的潜在危害和扩散波及面通常远大于漏洞 A（反射式）？

**48. 恶意代码的生态链：分类组合与实战威胁**
【场景背景】
在一份真实的 APT（高级持续性威胁）攻击溯源报告中，研究员详细描述了攻击链的流转过程：“攻击组织首先利用邮件投递了具备极强自我复制和网络传播能力的**蠕虫（Worm）**病毒组件；该组件在内网横向移动后，会在关键服务器中悄悄植入远控**木马（Trojan）**；最终，攻击者通过该远控通道下发指令，执行了破坏性的勒索**病毒（Virus）**模块。”
【问题】
虽然上述恶意代码概念在理论定义上各有侧重（如蠕虫强调自主网络传播，木马强调隐蔽伪装控制，病毒强调本地感染寄生），为什么在复杂的现代高级网络攻击行动中，安全防御团队通常将它们视作同一条战术利用链上的“组合武器模块”，而不是孤立的威胁实体进行割裂式的防守分析？

## 四、综合题（4题）

**49. 基础内存破坏与漏洞利用：栈溢出与防护绕过**
【场景背景】
某 32 位 Windows 应用程序中存在一个处理外部输入的函数，其源码如下：
```c
void vulnerable(char *input) {
    char buf[32];
    strcpy(buf, input);
}
```
程序编译时开启了 ASLR（地址空间布局随机化），但未开启 DEP（数据执行保护）和 Stack Canary。安全研究员已知该程序加载的某个特定 DLL 中存在 `system()` 函数，并且由于配置问题，该 DLL 的基址在每次运行时固定不变（即 ASLR 未对该 DLL 生效）。

【问题】
（1）**内存布局分析（L4）**：当程序调用 `vulnerable` 函数时，会在栈上建立独立的栈帧。请按从低地址到高地址的顺序，简述该栈帧中局部变量 `buf`、保存的旧栈底指针（saved EBP）以及返回地址（Return Address）这三者的相对排列位置。
（2）**溢出点计算（L4）**：攻击者已知 `buf` 大小为 32 字节，在 32 位程序中，攻击者至少需要输入多少个字节的填充数据，才能让下一个字节刚好开始覆盖栈上的返回地址？
（3）**控制流劫持（L6）**：攻击者计划利用已知的 `system()` 函数地址来劫持程序执行流。在 32 位 `__cdecl` 调用约定下，通过覆盖返回地址跳转到 `system()` 后，攻击者还需要在栈上如何布置数据，才能成功传递命令行参数？（请依次说明 system地址、伪造的返回地址、参数指针三者的排列顺序）。
（4）**防御机制对抗（L7）**：如果程序重新编译并开启了 Stack Canary 保护（Canary 值位于局部变量和返回地址之间），此时若直接进行超长字符串的线性溢出，会导致程序立即崩溃退出。为了成功绕过 Canary 劫持返回地址，攻击者必须提前利用另一种漏洞（如格式化字符串漏洞）从内存中获取到什么关键数据？

**50. 恶意代码植入与逆向工程：PE 注入与调试分析**
【场景背景】
在一次恶意软件分析中，安全研究员截获了一个被篡改的 Windows 可执行程序 `target.exe`。
已知该程序的 PE 镜像基址（ImageBase）为 `0x00400000`，原本的入口点（OEP）相对虚拟地址（RVA）为 `0x00003000`。
攻击者在该文件的空白代码节区注入了一段恶意汇编代码（主要功能为调用 `MessageBoxA` 弹窗勒索），注入代码的绝对虚拟地址（VA）起始于 `0x00405000`。

【问题】
（1）**PE 结构解析（L2）**：PE 加载器在执行程序前，需要知道从哪里开始执行第一条指令。请计算该注入代码的 RVA 是多少？攻击者在修改 PE 文件头时，应将 `AddressOfEntryPoint` 字段修改为何值，才能让程序一启动就执行恶意代码？
（2）**底层调用规约（L2）**：恶意代码需要调用系统的 `MessageBoxA` API。根据 Windows API 普遍采用的 `__stdcall` 调用约定，调用该函数时，底层的参数压栈顺序是**从左到右**还是**从右到左**？
（3）**隐蔽性控制（L2/L6）**：为了实现恶意行为的隐蔽性，不让受害者察觉程序已被篡改，注入的恶意代码在执行完弹窗操作后，必须通过什么操作（如特定的跳转指令），将控制权交还给哪个地址，才能保证原程序的原始业务逻辑继续正常运行？
（4）**动态调试验证（L3）**：如果分析人员使用 OllyDbg 或 x64dbg 动态调试该程序，载入后默认会停留在被篡改的入口点。为了验证注入的代码是否真正调用了目标 API，分析人员可以在系统库的 `MessageBoxA` 函数开头下达什么类型的断点进行拦截观察？

**51. 经典 Web 漏洞串联：从注入到任意代码执行**
【场景背景】
某老旧的 Web 业务系统在代码架构上存在多个独立的安全隐患，其三个主要功能模块及其对应的请求方式如下：
*   **模块 A（用户登录）**：`POST /login.php`，参数 `user` 和 `pwd` 直接被拼接到后端的 SQL 查询语句中。
*   **模块 B（头像上传）**：`POST /upload.php`，后端在处理上传文件时，仅仅检查了请求体中的 MIME 类型是否为 `image/jpeg`。
*   **模块 C（文件读取）**：`GET /page.php?file=xxx`，后端使用 `include()` 函数直接包含了参数传入的文件路径。

【问题】
（1）**身份劫持（L8）**：在模块 A 的登录功能中，攻击者可以在 `user` 字段输入什么样的经典“万能密码” Payload，使得拼接后的 SQL 语句 `WHERE user='...' AND pwd='...'` 逻辑永远返回真，从而无需密码直接登录管理员账号？
（2）**恶意载荷投递（L8）**：在模块 B 的上传功能中，由于后端只校验了 MIME 类型，攻击者可以通过抓包篡改 HTTP Header，将一个包含恶意 PHP 代码的文件（如扩展名为 `.jpg` 的一句话木马）伪装成合法图片上传至服务器。请简述这种绕过手段主要利用了 HTTP 协议的哪个字段？
（3）**攻击链闭环（L8）**：单独存在的恶意 `.jpg` 图片并不能自动执行。攻击者如何利用模块 C 的文件包含（LFI）漏洞，使得上述上传的伪装图片被服务器的 PHP 引擎作为代码进行解析和执行？
（4）**安全防御建设（L8）**：针对上述三个漏洞，请从防范的角度出发，分别写出一种最核心、最彻底的防御技术（例如针对 SQL 注入应使用什么机制？针对上传和包含应如何限制？）。

**52. 漏洞挖掘与自动化分析：高级测试技术论证**
【场景背景】
某专业安全审查团队拿到了一款未公开源码的闭源 Windows 桌面应用程序，该程序的核心功能是解析用户导入的自定义协议格式文件。团队需要综合运用各种软件安全分析技术，全面评估其是否存在内存破坏、逻辑缺陷或注入类漏洞。

【问题】
请根据团队遇到的以下四大分析场景，从**【模糊测试（Fuzzing）、污点分析、符号执行、程序切片】**这四种专业技术中选择最合适的一项填入括号内，以指导安全团队的后续工作：
（1）**快速发现崩溃（L7）**：团队希望在尽可能短的时间内，向该程序持续喂入海量的畸形变异文件，目的是通过监控程序的异常退出，快速探测出程序潜在的内存破坏类漏洞。首选技术：________。
（2）**数据流向追踪（L7）**：团队想确认从用户文件读取到的“非信任输入数据”，是否能够畅通无阻地顺着内存数据流，最终到达危险的系统执行函数（如 `system`）。首选技术：________。
（3）**深层约束求解（L7）**：分析人员发现程序的外层存在复杂的特定“魔术字节（Magic Byte）”和多重 `if-else` 校验逻辑，纯随机变异的输入始终被拦截在外层。为了计算出能走入深层特定代码分支的精确输入数据，自动求解条件约束。首选技术：________。
（4）**剥离冗余代码（L2/L7）**：在逆向分析某个关键变量时，代码极其冗长复杂。为了降低分析难度，团队希望在保持语义不变的前提下，将所有与该关键变量无关的干扰代码全部剔除，只保留核心逻辑。首选技术：________。
